# <a name="host-tools-for-frontrow-events"></a>FrontRow イベントのホスト ツール

この記事は、イベントに対して FrontRow が有効になっている場合に基づいて作成されます。 イベントに対して FrontRow を有効にするには、次の記事を参照してください。 FrontRow 機能を使用した対象ユーザー [のスケーリング](../faqs/scaling-audiences.md)。

FrontRow 以外のイベントのホスト ツールについては、ホスト ツールの概要に関 [するページを参照してください。](host-tools-overview.md)

イベントが FrontRow に設定されたので、FrontRow で有効になっているすべてのすばらしい機能を使用する方法を見てみます。

最初にイベントを作成するときに、そのイベントのホスト 'Role' を使用して設定しました。 また、イベントの実行を支援する他のユーザーにホスト ロールを与え、ホスト機能はすべて自分と同じものにすることもできます (ロールの詳細については、イベント、ワールド、グループ [、AltspaceVR](../getting-started/roles.md)のロールに関するページを参照してください)。 ホスト ロールを使用すると、ライブ仮想イベントを簡単かつ簡単にホストするために特別に作成された特別な機能にアクセスできます。

これらの機能、機能、および AltspaceVR で FrontRow イベントをホストする場合のその他の役立つヒントの詳細な概要を次に示します。

## <a name="host-tools-overview"></a>ホスト ツールの概要

ホスト ツールは、ビューのフィールドの右下に表示されます。 これは、イベントが作成された直後に使用できます。そのため、いつでもイベントを入力し (イベントのホストまたはモデレーターにはイベントの開始時刻は適用されません)、showtime の前にツールを使い始めることができます。 ホスト ツールを使用すると、オンエア (FrontRow イベントの実行に必要)、対象ユーザーのミュートとメッセージング、ステージのブロックのオンとオフの切り替え、自分のステージのブロックの切り替え、自分自身の "手を挙げる" などのクイック クリック アクセスが提供されます。ゲストは、質問やフィードバックを提供したり、FrontRow Room から Room に移動したりすることができます。

![使用可能なアクションが強調表示された [ホスト ツール] メニューが開きます](images/host-front-row-img-01.png)

*[GO ON-AIR] ボタンが表示されている場合は、イベントが FrontRow に設定されている必要があります。*

### <a name="toggle-mute-all"></a>切り替Mute-All

参加者をミュートすることで、気を散らす音を制限し、FrontRow イベントで物事を快適で落ち着かないようにすることができます。 ミュート関数は、ホストが許可している限り、またはホストがスペースを離れる限り、続く時間です。 参加者がイベントに入った場合、またはミュートを解除しようとすると、参加者は、ホストによってミュートされたと通知する画面のメッセージを受け取ります。 ホストがミュートを解除すると、参加者は、未変更も通知するメッセージを受け取ります。

### <a name="message-all"></a>メッセージすべて

ホストは、FrontRow イベント全体に存在するすべての参加者にメッセージを送信できます。 クイック ノートのいずれかを使用するか、元のメモを作成します。 送信すると、すべてのユーザーがポップアップとしてメッセージを受信します。

### <a name="amplify-my-voice"></a>自分の声を増幅する

ホストはマイクを増幅して、フロント 行イベントのすべてのユーザーが距離に関係なく、自分の言っていることを確実に聞き取る可能性があります。

### <a name="go-on-air"></a>Go On-Air

ホスト ロールを持つユーザーは、このボタンを表示できます。 [オンエアに移動] をクリックすると、すべての FrontRow ルームにアバターが "ミラー化" されます。 "ミラー化" が完了すると、FrontRow イベントのすべてのインスタンスに対して、自分の言い回しと実行のすべてが表示されます。

FrontRow イベント全体をオンエアすることが非常に重要です。 ホスト ツールがすべての FrontRow ルームで正しく動作するには、オンエアである必要があります。そうしないと、ホスト ツールは自分の部屋でのみ機能します。

> [!NOTE]
> イベントに再び参加する必要がある場合は、Campfire に戻り、インターネット接続を失い、Room から Room に移動します。戻ってくると、オンエアになる予定はありません。 戻った後、もう一度 [オンエアに移動] をクリックしてください。

### <a name="toggle-stage-blocker"></a>ステージ ブロッカーの切り替え

参加者が立ち上がる時間と場所を決定します。 スポットライトを設定したり、交代で発表したり、ステージ上の全員をグループ のダンス パーティーに招待したりします。

> [!IMPORTANT]
> すべての世界にステージ ブロッカーが含まれている場合はないので、イベント テンプレートを賢明に選択してください。 公式イベントを作成する前に、テスト イベントに関するイベント テンプレートをよく理解してください。

## <a name="participation-panel"></a>参加パネル
 
このパネルを使用すると、イベント参加者との対話に役立ちます。 Q&A を用意したり、オープン マイクに対して実行するキューを形成したり、話しているゲストもミラー化されているのを確認したりして、すべての FrontRow ルーム間で両方のユーザーが互いに話し合うのを全員が聞くことができることを確認できます。

![[参加] タブが選択されたホスト パネルが開きます](images/host-front-row-img-05.png)

参加パネルは、対象ユーザーからの対話と参加を管理するのに役立つ設計です。 ここでは、[Allow Hand Raises]/(ハンド レイズを許可する)オプションをオンに切り替え、対象ユーザーの参加を解除できます。 **FrontRow イベントでは、これをアクティブにするには、最初にオンエアである必要があります。それ以外の場合、このボタンは灰色で表示されます。**

[Allow Hand Raises]/(ハンド レイズを許可する)関数がアクティブになると、FrontRow イベントのすべてのユーザーに対して[Raise Hand]/(手を上げる)" ボタンが、そのフィールドオブビューの右下のセクションに表示されます。

![参加パネルで強調表示されているハンド関数を発生する](images/host-front-row-img-06.png)

*[Raise Hand]をクリックすると、制御するキューにゲストが入ります。*

ゲストは、このボタンをクリックすると、参加パネルに表示されます。

![キューで手を挙げたゲストが強調表示された参加パネル](images/host-front-row-img-07.png)

追跡を支援するために、参加パネルでは、ゲストが呼び出されるのを待っているときに赤い通知が表示され、各ゲストが待機している時間も通知されます。

[オンエア + Megaphone] ボタンをクリックすることで、これらのボタンを呼び出す必要があります。これにより、アクティブになるとボタンが青になります。 このアクションにより、ハンド レイザーに 2 つのメッセージが送信され、マイクのミュートが解除され、"ホストから電話が出され、音量が上がった" というメッセージが表示されます。 受け入れられると、ボタンが青に変わります。これにより、オンエアの "ミラー化" と 「私の声の増幅」の "Megaphoned" が表示され、聞こえます。

操作が完了したら、最初に青いボタンを再度クリックしてオフエアにします。ボタンは青ではなくなりました。次に、[X] をクリックして一覧から [閉じ] をクリックします。 これで、2 番など、人気の高い:)

また、一時的なステージ アクセス権を与えるか、右側の [すべて閉じてください] ボタンをクリックして、ハンド レイサーの一覧を完全にクリアし、新しく開始することもできます。

## <a name="room-panel"></a>[ルーム] パネル

FrontRow イベント中に Rooms を管理するために必要な情報/ボタンを簡単に確認できます。

![ホスト パネルから [ルーム] パネル タブが開きます](images/host-front-row-img-08.png)

***(You'is Here) と表示されている** のに注意してください。つまり、現在の部屋です。*

部屋には、領域のリセット、領域の削除、ユーザーの再配布、およびそこにテレポートを行うボタンがあります。

![部屋のパネルで強調表示されている [領域のリセット] ボタン](images/host-front-row-img-09.png)

現在の部屋で [領域のリセット] を実行します。 すべての部屋のスペースを一度にリセットすることはできません。リセットを行うには、各部屋に移動する必要があります。

![部屋のパネルで強調表示されている [スペースの削除] ボタン](images/host-front-row-img-10.png)

現在の Room は完全に削除されます。 削除されたルーム内のすべてのユーザーが自動的にイベントに再参加し、別のルームに参加します。

![部屋のパネルで強調表示されている [ユーザーの再配布] ボタン](images/host-front-row-img-11.png)

場合によっては、FrontRow イベントがしばらくの間オンにされた後、最初のいくつかのルームはほぼ容量に入っていますが、最後の Room または 2 では、Room あたりのゲストの数が少ない場合があります。 このボタンをクリックすると、その部屋のゲストは、現在の部屋に残る、またはより多くの設定された部屋に移動するオプションが表示されます。

> [!NOTE]
> 再配布するルームに移動する必要は、ルーム パネルから任意のルームの [再配布ユーザー] ボタンをクリックできます。 同じ部屋に何度も行いすぎずに、そのゲストは少数の人と一緒にいたいだけであり、部屋の移動を求めるメッセージを受け取り続けたくない場合があります。

![部屋のパネルで強調表示されている [Teleport there] ボタン](images/host-front-row-img-12.png)
