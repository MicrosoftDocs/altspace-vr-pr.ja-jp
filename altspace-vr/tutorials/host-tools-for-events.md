# <a name="host-tools-for-frontrow-events"></a>FrontRow イベントのホスト ツール

この記事は、イベントに対して FrontRow が有効になっている場合に基づいて作成されます。 イベントに対して FrontRow を有効にするには、次の記事を参照してください。 FrontRow 機能を使用した対象ユーザー [のスケーリング](../faqs/scaling-audiences.md)。

FrontRow 以外のイベントのホスト ツールについては、ホスト ツールの概要に関 [するページを参照してください。](host-tools-overview.md)

イベントが FrontRow に設定されたので、FrontRow で有効になっているすべてのすばらしい機能を使用する方法を見てみます。

最初にイベントを作成するときに、そのイベントのホスト 'Role' を使用して設定されました。 また、イベントの実行を支援する他のユーザーにホスト ロールを与え、ホスト機能はすべて自分と同じものにすることもできます (「Roles [in Events, Worlds, Groups and in AltspaceVR](../getting-started/roles.md)」(イベント、ワールド、グループ、AltspaceVR のロール) を参照してください)。 ホスト ロールを使用すると、ライブ仮想イベントを簡単かつ簡単にホストするために特別に作成された特別な機能にアクセスできます。

これらの機能、機能、および AltspaceVR で FrontRow イベントをホストする場合のその他の役立つヒントの詳細な概要を次に示します。

## <a name="host-tools-overview"></a>ホスト ツールの概要

ホスト ツールは、ビューのフィールドの右下に表示されます。 これは、イベントが作成された直後に使用できます。そのため、いつでもイベントを入力し (イベントのホストまたはモデレーターにはイベントの開始時刻は適用されません)、showtime の前にツールを使い始めることができます。 ホスト ツールを使用すると、オンエア (FrontRow イベントの実行に必要)、対象ユーザーのミュートとメッセージング、ステージのブロックのオンとオフの切り替え、自分のステージのブロックの切り替え、自分自身の "手を挙げる" などのクイック クリック アクセスが提供されます。ゲストは、質問やフィードバックを提供したり、FrontRow Room から Room に移動したりすることができます。

![使用可能なアクションが強調表示された [ホスト ツール] メニューが開きます](images/host-front-row-img-01.png)

*[GO ON-AIR] ボタンが表示されている場合は、イベントが FrontRow に設定されている必要があります。*

### <a name="toggle-mute-all"></a>切り替Mute-All

参加者をミュートすることで、気を散らす音を制限し、FrontRow イベントで物事を快適で落ち着かないようにすることができます。 ミュート関数は、ホストが許可している限り、またはホストがスペースを離れる限り、続く時間です。 参加者がイベントに参加するか、ミュートを解除しようとすると、参加者は、ホストによってミュートされたと通知する画面のメッセージを受け取ります。 ホストがミュートを解除すると、参加者は、未変更も通知するメッセージを受け取ります。

### <a name="message-all"></a>メッセージすべて

ホストは、FrontRow イベント全体に存在するすべての参加者にメッセージを送信できます。 クイック ノートのいずれかを使用するか、元のメモを作成します。 送信すると、すべてのユーザーがポップアップとしてメッセージを受信します。

### <a name="amplify-my-voice"></a>自分の声を増幅する

ホストはマイクを増幅して、フロント 行イベントのすべてのユーザーが距離に関係なく、自分の言っていることを確実に聞き取る可能性があります。

### <a name="go-on-air"></a>Go On-Air

ホスト ロールを持つユーザーは、このボタンを表示できます。 [オンエアに移動] をクリックすると、すべての FrontRow ルームにアバターが "ミラー化" されます。 "ミラー化" が完了すると、FrontRow イベントのすべてのインスタンスに対して、自分の言い回しと実行のすべてが表示されます。

FrontRow イベント全体をオンエアすることが非常に重要です。 ホスト ツールがすべての FrontRow ルームで正しく動作するには、オンエアである必要があります。そうしないと、現在の部屋でのみ機能します。

> [!NOTE]
> イベントに再び参加する必要がある場合は、Campfire に戻り、インターネット接続を失い、Room から Room に移動します。戻ってくると、オンエアになる予定はありません。 戻った後、もう一度 [オンエアに移動] をクリックしてください。

### <a name="toggle-stage-blocker"></a>ステージ ブロッカーの切り替え

参加者が立ち上がる時間と場所を決定します。 スポットライトをホグしたり、交代で発表したり、ステージ上の全員をグループのダンス パーティーに招待したりします。

> [!IMPORTANT]
> すべての世界にステージ ブロッカーが含まれている場合はないので、イベント テンプレートを賢明に選択してください。 公式イベントを作成する前に、テスト イベントに関するイベント テンプレートをよく理解してください。
